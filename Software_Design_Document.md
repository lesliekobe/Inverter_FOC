## 1. 文档概述

### 1.1 目的与范围

本文档描述低压三相矢量变频器（0.75kW～75kW）软件系统的完整设计方案，涵盖架构设计、模块接口、算法原理、数据流、调度时序等内容。

目标读者：嵌入式软件工程师、电机控制算法工程师、硬件工程师、系统测试工程师。

### 1.2 产品规格

| 项目 | 规格 |
|------|------|
| 功率范围 | 0.75kW ～ 75kW |
| 额定电压 | 三相380V AC（允许波动±15%） |
| 输出频率 | 0.00Hz ～ 400.00Hz |
| 载波频率 | 2kHz ～ 16kHz（默认10kHz） |
| 控制模式 | V/F开环 / 矢量闭环（FOC）/ PID恒压 / 多段速 |
| 适配电机 | 异步电机（IM）/ 永磁同步电机（PMSM） |
| 主控芯片 | STM32F4/F7/H7（ARM Cortex-M4/M7）或 TI DSP28335 |
| 通讯接口 | RS-485 Modbus RTU（默认115200 8N1） |
| 保护功能 | 过流/过压/欠压/过热/过载/堵转/缺相 |
| 参数存储 | 片内Flash或外置EEPROM（掉电不丢失） |
| 编码器 | ABZ增量编码器（可选） |

## 2. 系统架构

### 2.1 分层架构总览

软件系统采用**单向依赖、分层隔离**的架构设计，自上而下共6层：

```
┌────────────────────────────────────────────────────────────────────┐
│                         HMI 层（人机交互）                           │
│   职责：按键扫描、数码管/LCD显示、参数编辑、状态轮播                   │
│   调用周期：10ms                                                    │
├────────────────────────────────────────────────────────────────────┤
│                         APP 层（应用逻辑）                           │
│   职责：加减速控制、多段速逻辑、PID工艺调节、DI/DO逻辑、Modbus指令解析 │
│   调用周期：1ms                                                     │
├────────────────────────────────────────────────────────────────────┤
│                      MC 层（电机控制核心）                           │
│   职责：FOC矢量控制、V/F控制、SVPWM调制、ADC采样、保护判断             │
│   调用周期：10kHz（电流环）、1kHz（速度环）                          │
├────────────────────────────────────────────────────────────────────┤
│                    DRV 层（外设驱动抽象）                            │
│   职责：PWM输出、ADC采样、GPIO输入输出、UART通讯、编码器计数           │
│   接口方式：API函数调用                                             │
├────────────────────────────────────────────────────────────────────┤
│                    BSP 层（底层硬件抽象）                            │
│   职责：时钟配置、GPIO模式、中断向量表、芯片寄存器初始化               │
├────────────────────────────────────────────────────────────────────┤
│                    OS 层（调度内核）                                 │
│   职责：裸机分时调度 或 FreeRTOS任务管理                              │
└────────────────────────────────────────────────────────────────────┘
```

**依赖规则**：每层只能调用下层接口，禁止反向调用或跨层调用（单向依赖）。

## 3. 全局公共层（Global）

### 3.1 全局上下文结构体（GlobalCtx）

全局上下文 `g_ctx` 是贯穿所有模块的"数据中枢"，所有层共享同一个实例：

```c
typedef struct {
    uint16_t    run_state;         // 当前运行状态
    uint16_t    fault_code;        // 当前故障码
    uint16_t    run_mode;          // 控制模式（VF/FOC/PID/多段速）

    ParamGroup  param;             // 参数组（EEPROM存储）
    MotorCtrlOut motor_out;        // 电机算法输出
    AdcRawData  adc_raw;           // ADC原始采样值
    IOState     io_st;             // DI/DO状态
    CommState   comm_st;           // 通讯状态
    ProtectFlag prot_flag;         // 保护阈值标记
} GlobalCtx;
```

### 3.2 参数组（ParamGroup）

参数组包含全部可配置参数，按功能分为5个子组：

| 参数子组 | 参数项数 | 存储位置 | 说明 |
|----------|----------|----------|------|
| 基础参数 | 16 | EEPROM | 电机额定值、V/F曲线、载波频率 |
| 保护参数 | 16 | EEPROM | 过流/过压/欠压/过温阈值 |
| 通讯参数 | 7 | EEPROM | Modbus地址/波特率/超时 |
| 工艺参数 | 13 | EEPROM | PID增益、目标值、多段速预设 |
| 状态统计 | 8 | RAM | 运行时间、故障次数等只读统计 |

### 3.3 故障管理（FaultMgr）

故障管理采用**环形缓冲区**存储历史故障记录：

- 容量：32条（FIFO循环覆盖）
- 每条记录包含：故障时间戳、故障码、发生时转速、母线电压、输出电流
- 故障锁定机制：故障发生后，状态锁死，需手动复位或断电重启
- 故障分类：
  - **瞬时保护**：过流、外部急停 → 立即封锁PWM
  - **反时限保护**：I²t过载曲线（120%额定电流持续60s跳闸）
  - **慢性保护**：温升累积、温度阈值

## 5. DRV层（Driver）

### 5.1 PWM驱动（drv_pwm）

**功能**：三相互补PWM输出，死区生成，PWM使能/封锁。

**关键配置**：
- 定时器：TIM1（高级定时器，支持互补输出+死区）
- 计数模式：中心对齐模式1（Up-Down，单次下溢更新）
- 死区时间：~100ns（DTG=0x45，对应约2个HCLK周期）
- PWM频率：`f_PWM = SYSCLK / ((ARR+1) * 2)`
  - 默认10kHz：ARR = 168000000 / 10000 / 2 - 1 = 8399

**安全机制**：
- 任何故障发生时，`DRV_PWM_Disable()` 将MOE位清零，三相输出强制为低电平
- 故障恢复后需手动重新使能

### 5.2 ADC驱动（drv_adc）

**功能**：6通道同步采样，DMA循环搬运。

**采样通道序列**：
```
[ Ia (PA0) | Ib (PA1) | Udc (PA2) | NTC (PA3) | AI1 (PA4) | AI2 (PA5) ]
```

**采样时序**：
- 采样时间：479个ADC时钟周期（确保充放电稳定）
- DMA循环模式：每采样完成自动传输到内存
- 采样触发：TIM1_CC触发（与PWM周期同步）

**物理量换算**：
- 电流：ACS712-20A电流传感器 → 100mV/A → ADC零点2048
- 电压：分压电阻47kΩ+100Ω → 衰减比约0.00212 → 540V对应~1.14V
- NTC温度：B=3950热敏电阻，10kΩ@25℃，通过Beta方程换算温度

### 5.3 UART驱动（drv_uart）

**功能**：RS-485 Modbus RTU通讯，收发环形缓冲区。

**关键设计**：
- 收发独立缓冲区（256字节环形buffer）
- TXE中断：发送缓冲区数据，同时控制RS-485 DE引脚
- RXNE中断：接收字节存入缓冲区
- DE引脚控制：发送前拉高（使能驱动），发送完毕自动拉低（恢复接收）
- 波特率支持：4800 / 9600 / 19200 / 38400 / 115200

### 5.4 编码器驱动（drv_encoder）

**功能**：ABZ正交编码器脉冲计数，速度测量。

**关键设计**：
- 定时器TIM4配置为编码器模式（TI1FP1 + TI2FP2）
- 4倍频：每相每个周期产生4个计数（提升分辨率）
- 速度测量：M法（固定时间间隔数脉冲数）
- 滑动平均滤波：8点窗口滤波，抑制脉冲计数抖动
- Z脉冲：每转一次，用于零点定位和圈数计数

## 6. MC层（Motor Control）

### 6.1 SVPWM空间矢量调制

#### 6.1.1 算法原理

SVPWM将三相电压投影到αβ坐标系，得到空间电压矢量，在正六边形的6个扇区内用7段式零矢量插入法合成目标矢量：

```
α
 ↑
 │      U2(010)
 │       ↗
 │      /  U3(011)
 │     /  ↗
 │  U1(110)   U0(100)→──────→ β
 │ ↗         ↗
 │/           U5(101)
U4(001)       U6(111)零矢量
```

#### 6.1.2 扇区判断

根据Vα和Vβ的符号和关系判断扇区：

```c
a = Vβ
b = √3/2·Vα - 0.5·Vβ
c = -√3/2·Vα - 0.5·Vβ

if (a≥0) sector += 1
if (c≥0) sector += 2
if (b<0) sector += 4
// sector: 1~6
```

#### 6.1.3 矢量作用时间计算

以扇区1为例，U0(100)和U1(110)作用时间：

```
T1 = √3/6 · T_PWM · (√3·Vα - Vβ) / (Udc/2)
T2 = √3/6 · T_PWM · (2·Vβ) / (Udc/2)
T0 = T_PWM - T1 - T2
```

七段式插入顺序（扇区1）：`T0/4 → T1/2 → T2/2 → T7 → T2/2 → T1/2 → T0/4`

#### 6.1.4 过调制处理

当电压矢量超出内切圆半径 `Udc/√3` 时，等比例缩放Vα、Vβ，保证输出不饱和：

```
v_mag = sqrt(Vα² + Vβ²)
if (v_mag > Udc/√3) {
    Vα *= (Udc/√3) / v_mag
    Vβ *= (Udc/√3) / v_mag
}
```

#### 6.1.5 关键代码接口

```c
SVPWM_Init(&g_svpwm, arr_period, Udc);  // 初始化（main中调用）
SVPWM_Calc(&g_svpwm, Valpha, Vbeta, &out); // 计算（10kHz中断中调用）
// 输出：out.CCR1/CCR2/CCR3 → TIM1通道比较寄存器
```

### 6.2 FOC矢量控制

#### 6.2.1 坐标变换链路

```
三相电流 Ia, Ib, Ic
      ↓ Clarke变换
αβ坐标系 Iα = Ia
              Iβ = (Ia + 2Ib)/√3
      ↓ Park变换（旋转坐标系）
dq坐标系 Id = Iα·cosθ + Iβ·sinθ
             Iq = -Iα·sinθ + Iβ·cosθ
      ↓ 电流环PI（Id_ref=0, Iq_ref=转矩计算值）
dq电压   Vd, Vq
      ↓ 反Park变换
αβ坐标系 Vα = Vd·cosθ - Vq·sinθ
             Vβ = Vd·sinθ + Vq·cosθ
      ↓ SVPWM
三相PWM占空比
```

#### 6.2.2 电流环PI控制器

离散PI控制器，含积分限幅和输出限幅：

```c
// 离散PI（含防积分饱和）
p_term = Kp * err
i_term += Ki * err * Ts
if (i_term > out_max*0.8) i_term = out_max*0.8
if (i_term < -out_max*0.8) i_term = -out_max*0.8
output = p_term + i_term
output = LIMIT(output, -out_max, out_max)
```

#### 6.2.3 速度环PI控制器

速度环1kHz调用，输出Iq_ref（转矩电流给定）：

```
speed_err = speed_ref - speed_fb
Iq_ref = PI_Control(speed_err, Kp=0.08, Ki=0.001, out_max=1.2×I_rated)
```

弱磁控制：当转速超过基速110%时，Id_ref负向减小，进行弱磁升速。

#### 6.2.4 角度获取

- **有编码器**：直接读取编码器计数器，换算电角度 `θ = position × 2π / (4×PPR)`
- **无编码器**：滑模观测器（SMO）或直接估算法（适用于V/F模式）

### 6.3 V/F开环控制

V/F模式用于异步电机，无需编码器：

```
V = V_rated × (f / f_rated) + V_boost
// 低频时叠加补偿电压V_boost（启动转矩提升）
// 默认V_boost = 20V（当f<10Hz时线性衰减至0）
```

角度积分：`θ += 2π × f × Ts`（电气角速度积分）

### 6.4 保护判断（1ms任务）

| 保护项 | 触发条件 | 动作 |
|--------|----------|------|
| 瞬时过流 | Imax > I_max（1.5×额定） | 立即封锁PWM |
| 过压 | Udc > 750V | 封锁PWM |
| 欠压 | Udc < 350V | 封锁PWM |
| IGBT过温 | NTC_temp > 85°C | 封锁PWM |
| 反时限过载 | ∫(I²/I_rated² - 1.2²)dt / 60s > 1 | 封锁PWM |
| 堵转 | speed≈0 且 Iq > 0.8×I_rated 持续2s | 封锁PWM |

## 7. APP层（Application Logic）

### 7.1 加减速控制（app_speed）

**频率给定优先级**（高→低）：
```
Modbus通讯 > 多段速端子 > AI模拟量 > 面板给定
```

**斜坡曲线**：线性加减速，支持可配置加减速时间（默认5s/5s）：

```
f_current += sign(f_target - f_current) × (f_rated / accel_time) × Ts
```

### 7.2 PID工艺调节器（app_pid）

离散PID，含微分滤波和输出限幅：

```
P = Kp × err
I += Ki × err × Ts（积分限幅）
D = Kd × (err - err_prev) / Ts（微分滤波）
output = P + I + D（输出限幅）
err_prev = err
```

典型应用：**恒压供水**：压力反馈（AI1 4-20mA）→ PID计算 → 叠加到频率给定

### 7.3 DI/DO逻辑（app_io_logic）

**数字输入处理**（上升沿检测）：
- DI0（启动）：run_cmd_pending
- DI1（停止）：stop_cmd_pending
- DI2（反转）：方向切换
- DI3（外部故障）：FAULT_EXTERNAL
- DI4~6（多段速）：组合选择8段预设频率
- DI7（急停）：FAULT_EMERGENCY（最高优先级）

**数字输出控制**：
- DO0（运行接触器）：运行中且无故障时闭合
- DO1（故障输出）：故障锁定时闭合（继电器失电表示故障）
- DO2（散热风扇）：温度>40°C 或 运行中>35°C 时闭合
- DO3（制动）：转速下降时闭合（能耗制动）
- DO4/DO5（LED）：运行/故障状态指示

### 7.4 Modbus RTU指令处理（app_modbus_cmd）

**支持功能码**：

| 功能码 | 名称 | 说明 |
|--------|------|------|
| 0x03 | Read Holding Registers | 读保持寄存器 |
| 0x06 | Write Single Register | 写单个寄存器 |
| 0x10 | Write Multiple Registers | 写多个寄存器 |

**寄存器映射**：

| 地址 | 名称 | 读写 | 说明 |
|------|------|------|------|
| 0x0000 | CMD | RW | 1=启动，0=停止 |
| 0x0001 | FREQ_REF | RW | 频率给定，0.01Hz单位（5000=50Hz）|

## 8. HMI层（Human Machine Interface）

### 8.1 按键扫描（hmi_key）

## 9. 调度层（Scheduler）

### 9.1 裸机分时调度器（方案A，适用于中低端产品）

**时基**：SysTick或TIM2，1ms周期中断。

**任务列表**（优先级0最高→8最低）：

| 任务 | 周期 | 优先级 | 说明 |
|------|------|--------|------|
| MC_PROTECT | 1ms | 0 | 保护判断 |
| APP_SPEED | 1ms | 1 | 加减速斜坡 |
| APP_IO | 1ms | 2 | DI/DO逻辑 |
| MC_SPEED | 10ms | 5 | FOC速度环 |
| APP_MODBUS | 5ms | 3 | Modbus收发处理 |
| HMI_KEY | 10ms | 4 | 按键扫描 |
| MC_SAMPLE | 100ms | 6 | ADC采样同步 |
| HMI_DISP | 100ms | 7 | 显示刷新 |
| PARAM_SAVE | 100ms | 8 | 参数存储检查 |

**调度逻辑**：
```c
// main函数中循环调用
void Scheduler_Run(void) {
    while (1) {
        now = Scheduler_GetTick();
        for (each task in priority order) {
            if (now - task.last_tick >= task.period_ms) {
                task.last_tick = now;
                task.func();
            }
        }
        IWDG_Feed();  // 喂狗
    }
}
```

### 9.2 FreeRTOS方案（方案B，适用于高端产品）

**任务列表**：

| 任务 | 优先级 | 周期 | 堆栈 | 说明 |
|------|--------|------|------|------|
| vTask_MotorControl | 9（最高） | 1ms | 512W | FOC电流环+PWM |
| vTask_AppLogic | 8 | 1ms | 256W | 速度控制+IO |
| vTask_CommModbus | 6 | 5ms | 256W | 通讯处理 |
| vTask_HMI | 4 | 10ms | 256W | 显示+按键 |
| vTask_SlowMonitor | 2 | 100ms | 256W | 温度+存储 |

**任务间通信**：
- 速度给定：队列 `xQueueSpeedRef`
- 故障事件：队列 `xQueueFault`
- 参数访问：互斥锁 `xMutexParam`

## 12. 通讯协议

### 12.1 Modbus RTU帧格式

```
| 从机地址(1B) | 功能码(1B) | 数据(nB) | CRC16(2B) |
|------------|-----------|---------|-----------|
```

### 12.2 读保持寄存器（0x03）

**请求**：
```
| ADDR | 0x03 | 起始高(1B) | 起始低(1B) | 数量高(1B) | 数量低(1B) | CRC |
```

**响应**：
```
| ADDR | 0x03 | 字节数 | 数据... | CRC |
```

### 12.3 写单个寄存器（0x06）

**请求**：
```
| ADDR | 0x06 | 寄存器高 | 寄存器低 | 值高 | 值低 | CRC |
```

**响应**：原帧回显

### 12.4 写多个寄存器（0x10）

**请求**：
```
| ADDR | 0x10 | 起始高 | 起始低 | 数量高 | 数量低 | 字节数 | 数据... | CRC |
```
